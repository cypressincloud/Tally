# 分期付款功能需求

## 概述
在资产模块中新增"分期"功能，允许用户管理分期付款账户，跟踪每期还款状态。

## 需求详情

### 1. UI 改造
#### 1.1 添加资产对话框
- **当前**：使用 RadioGroup 单选框选择类型（资产/负债/借出/理财）
- **改为**：使用 Spinner 下拉框选择类型
- **新增选项**：分期（Installment）
- **样式参考**：项目中现有的下拉框样式（如分类选择、币种选择等）

#### 1.2 分期输入表单
当用户选择"分期"类型时，显示以下字段：
- **分期对象**（必填）：String，例如"花呗"、"信用卡"
- **总期数**（必填）：Int，例如 12 期
- **每期金额**（必填）：Double，例如 500.00
- **自动计算**：总金额 = 总期数 × 每期金额

### 2. 数据模型

#### 2.1 AssetAccount 扩展
需要在 `AssetAccount` 实体中新增字段：
```java
// type = 4 表示分期
public int type; // 0:资产, 1:负债, 2:借出, 3:理财, 4:分期

// 分期相关字段（仅当 type == 4 时使用）
public int totalInstallments;      // 总期数
public double installmentAmount;   // 每期金额
public String paidInstallments;    // 已还期数（JSON 数组，如 "[1,3,5]"）
```

#### 2.2 数据库迁移
需要升级数据库版本，添加新字段。

### 3. 分期卡片显示

#### 3.1 列表卡片（在负债列表中）
显示内容：
- **标题**：分期对象名称
- **副标题**：剩余期数/总期数（如 "还剩 8/12 期"）
- **金额**：每期金额（如 "¥500.00/期"）
- **总金额**：总金额（如 "总计 ¥6000.00"）
- **进度条**：可选，显示还款进度

#### 3.2 卡片样式
- 使用独特的图标或颜色标识分期类型
- 与普通负债卡片区分开

### 4. 分期详情页面

#### 4.1 页面布局
- **顶部信息区**：
  - 分期对象名称
  - 总期数、每期金额、总金额
  - 已还期数、剩余期数
  - 已还金额、剩余金额
  
- **期数网格**：
  - 使用 RecyclerView + GridLayoutManager
  - 每个期数显示为圆角矩形
  - 中间显示期数（如 "1"、"2"、"3"...）
  - **未还状态**：浅灰色背景
  - **已还状态**：主题色背景
  - 点击切换状态

- **底部操作区**：
  - 编辑按钮
  - 删除按钮

#### 4.2 交互逻辑
- 点击期数卡片：切换该期的还款状态
- 状态切换时：
  - 更新 `paidInstallments` 字段
  - 更新顶部统计信息
  - 触发 WebDAV 自动同步
- 删除分期：弹出确认对话框

### 5. 计入总资产逻辑
- 分期账户默认**不计入总资产**
- 用户可以选择是否计入
- 如果计入，按照负债逻辑处理（减少总资产）
- 计算金额 = 剩余期数 × 每期金额

### 6. 数据统计
- 在"我的负债"面板中，分期账户的金额 = 剩余期数 × 每期金额
- 总资产计算时，如果分期计入总资产，则减去剩余应还金额
